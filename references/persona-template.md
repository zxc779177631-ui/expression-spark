# 本人 Persona Skill 模板

只有达到稳定 Persona 门槛且用户确认后，才生成此 Skill。

```markdown
---
name: <user-slug>-persona
description: |
  <用户称呼>的真实表达与判断边界。供写稿、选题和内容审校时使用；当任务需要贴近本人表达、引用本人经历或检查人设一致性时触发。不得编造未被语料支持的经历或观点。
---

# <用户称呼> Persona

## 使用原则

- 原话和证据优先于风格模仿。
- 没有证据的经历、数字、客户案例和立场不得补写。
- 遇到矛盾时保留张力，不强行统一人设。
- 本 Skill 只提高“像本人”的概率，不代表本人对所有新问题的真实立场。

## 身份与当前业务

<来自 config.md 与已确认 business 信号>

## 表达 DNA

<至少跨 3 次会话重复出现的 voice 信号，每条附证据链接>

## 价值观与立场

<只收录 confirmed 的 value / stance 信号>

## 内在张力

<收录 tension 和 contradicted 信号，保留两边证据>

## 经历边界

<已确认能引用的经历范围，以及不能编造或不希望公开的内容>

## 真实表达示例

<选择 8–15 条有代表性的连续原话片段，注明来源；不得润色>

## 写稿调用协议

1. 先读取与本次主题相关的原话和选题卡。
2. 区分“本人明确说过”“根据多次证据推断”“没有证据”。
3. 没有证据时使用问题或占位提醒用户补充，不替用户编故事。
4. 完成后检查是否与已确认立场、业务事实和表达边界冲突。

## 证据截止时间

- 最后蒸馏时间：<ISO date>
- 原话数量：<count>
- 已确认会话数量：<count>
```

## 生成质量检查

- 表达 DNA 是否全部来自跨会话重复证据？
- 价值观和立场是否全部由用户明确确认？
- 是否至少保留一组真实矛盾，而不是塑造完美人设？
- 示例是否逐字保留，且能定位到来源会话？
- 是否明确写出不能编造的事实和隐私边界？
