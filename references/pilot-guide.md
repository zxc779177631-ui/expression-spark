# 小批用户试用指南

首轮邀请 5–8 名缺少现成语料、但确实想做个人内容的用户。每位用户至少完成 3 次表达会话，再判断 skill 是否有效。

## 试用设置

- 三种默认模式尽量均匀分配。
- 用户可以随时切换模式；记录切换原因。
- 鼓励使用豆包输入法、微信输入法或 Typeless 口述输入。
- 不要求用户必须产出视频，先验证是否愿意表达、能否沉淀真实素材。

## 每次会话记录

| 指标 | 记录方式 |
|---|---|
| 是否被聊开 | 用户自评 1–5 分 |
| 首个具体故事出现时间 | 从开始到首次出现人物、场景、选择和结果的分钟数 |
| 经确认原话数量 | `status` 中本次新增数量 |
| 轻量选题卡数量 | 本次确认保存数量 |
| 错误画像数 | 用户明确说“不像我/总结错了”的信号数量 |
| 模式切换 | 从哪种切到哪种，以及原因 |
| 再次使用意愿 | 用户自评 1–5 分 |

## 三次体验后的访谈问题

一次只问一个：

1. 哪一次你最容易说下去？当时 Agent 做了什么？
2. 哪个问题让你突然不想说了？
3. 审阅卡里哪些内容最像你，哪些最像 AI 总结？
4. 你愿不愿意把这些素材交给另一个 Agent 写稿？为什么？
5. 如果下周只能保留一种模式，你会选哪个？

## Persona 盲选测试

达到稳定 Persona 门槛后，用同一主题生成两版内容：

- A：普通写稿，不读取 Persona 或原话资产。
- B：读取 Persona 和相关 `context` 资产。

隐藏版本来源，请用户和一位熟悉用户的人分别判断：

- 哪版更像本人？
- 哪版包含编造或越界？
- 哪一句最有本人辨识度？

## 首轮成功标准

- 至少 60% 用户完成 3 次会话。
- 至少 70% 会话沉淀 3 条以上经确认原话。
- 错误画像率低于 15%。
- Persona 辅助文案在盲选中“更像本人”的比例高于普通文案。
- 用户没有发现未经确认保存、原话被改写或遗忘请求未被执行的情况。

## 需要优先修复的失败信号

- 用户回答越来越短：问题过于抽象、一次问太多或过早分析。
- 用户常选“只保存原话”：选题卡或画像提炼不可信。
- 错误画像集中在前两次会话：需要继续提高稳定信号门槛。
- 内容教练模式产出很多选题但用户不愿再聊：内容判断压过了表达体验。
- Persona 文案更像“人设模板”而不是本人：减少风格规则，增加真实原话与边界。
